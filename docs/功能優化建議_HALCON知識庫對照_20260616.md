## 功能新增建議

### 🟢 A. Ellipse 擬合（最貼合現有架構，CP 值最高）

- **知識庫依據**: `XLD/Features/fit_ellipse_contour_xld.txt`
- **現況**: 專案有 Circle、Line 擬合，**沒有 Ellipse**。
- **為何值得**: 閃測儀量測傾斜圓孔、橢圓特徵時，用圓擬合會有系統誤差。`fit_ellipse_contour_xld` 回傳 `Row/Column/Phi/Radius1/Radius2/StartPhi/EndPhi/PointOrder`。
- **工成**: **低**。完全套用 CLAUDE.md 已記載的「feature adapter 五件套」（Domain DTO + Application 介面 + Halcon adapter + test + WinForms wiring），與 CircleFitting 幾乎平行，可複製改寫。
- **注意**: 演算法名與 circle 不同（`fitzgibbon` / `fhuber` / `ftukey`），參數順序亦不同，須照離線 reference 核對。

### 🟢 B. Rectangle2 擬合

- **知識庫依據**: `XLD/Features/fit_rectangle2_contour_xld.txt`、`Applications/Measuring-2D/measure_stamping_part.txt`
- **現況**: 沒有矩形擬合。
- **為何值得**: 量測矩形件 / 沖壓件 / 方孔的長寬、角度與邊緣偏差，回傳 `Row/Column/Phi/Length1/Length2/PointOrder`。範例另示範以 `dist_rectangle2_contour_points_xld` 檢查邊緣凸起瑕疵——對閃測「輪廓度檢測」很實用。
- **工成**: **低**，同樣套用 feature adapter 模式。

### 🟡 C. 2D Metrology Model（架構級，威力最大但工成高）

- **知識庫依據**: `2D-Metrology/apply_metrology_model.txt`、`Applications/Measuring-2D/apply_metrology_model_diamond.txt`、`pm_measure_board.txt`
- **現況**: 專案是「自己把 edge detection 跟 fitting 串起來」；HALCON 的 metrology model 把整條 pipeline 包成一個物件。
- **為何值得**: 一次定義 circle/line/rectangle/ellipse 物件 → 自動佈 measure region、sub-pixel 邊緣、robust 擬合、outlier 剔除、`min_score`、`measure_transition='uniform'`（極性一致性檢查）。一張影像可同時量多個特徵、多 instance。是 HALCON 官方推薦的正規量測做法，比手動串接更穩、可重現（可存成 recipe）。
- **工成**: **高**。是新的一層抽象，需新增一組模型管理（Domain/Application/Halcon）。建議當成獨立里程碑，勿與小修混做。
- **誠實提醒**: 與現有 Circle/Line adapter 功能**部分重疊**。導入前須先決定「並存」或「逐步遷移」，否則會變成兩套量測路徑。

### 🟡 D. 相機校正 / 世界座標（精度天花板）

- **知識庫依據**: `Calibration/Rectification/image_to_world_plane.txt`、`lens_distortions.txt`、`Calibration/Multi-View/calibrate_cameras_*.txt`
- **現況**: `DistanceMeasurementParameters` 用**固定** `PixelSizeUmX/Y = 10.0` 線性換算 px→um。
- **問題**: 固定比例假設「無鏡頭畸變、無透視、整個視場放大率一致」。實際鏡頭邊緣有畸變、非遠心鏡頭有透視，導致視場邊緣量測誤差。對標榜精度的閃測儀，這是**目前的精度上限**。
- **為何值得**: 以標定板做相機校正 + `image_to_world_plane` 後，px→mm 為逐點正確的物理映射。
- **工成**: **高**，且需標定板硬體流程。屬「若客戶要求高精度才做」。

## 優化建議

### 🔸 E. 背光/剪影影像可先 threshold 縮小 `edges_sub_pix` 的 domain

- **知識庫依據**: `XLD/Features/fit_rectangle2_contour_xld.txt` 第 16–23 行明示「Since the metal part is backlit, ... constructing a ROI as small as possible ... increases processing speed significantly」（`fast_threshold` + `boundary` + `dilation_rectangle1` + `reduce_domain`）。
- **現況**: 專案 `DetectEdgesSubPix` 已用 ROI 矩形 `reduce_domain`，但 ROI 內仍整塊掃描。
- **評估**: 閃測本質為背光剪影，若單一 ROI 內邊緣稀疏，先 threshold 取邊界帶能再加速。但**收益視影像而定**且增加複雜度——建議**先量測現有效能瓶頸再決定**，不要為優化而優化。屬「有實測數據再做」。

## 優先順序建議

| 建議 | 類型 | 價值 | 工成 | 推薦 |
|------|------|------|------|------|
| A. Ellipse 擬合 | 功能 | 中高 | 低 | ⭐ **先做**，CP 值最高 |
| B. Rectangle2 擬合 | 功能 | 中高 | 低 | ⭐ 接著做 |
| C. Metrology Model | 功能 | 高 | 高 | 規劃為獨立里程碑，先討論定位 |
| D. 相機校正 | 精度 | 高 | 高 | 視精度需求，需硬體配合 |
| E. 背光 threshold 加速 | 優化 | 視情況 | 低中 | 先測效能再決定 |

**最務實的起手式是 A（Ellipse 擬合）**——完全落在 CLAUDE.md 已定義的 feature adapter 模式內，是「最小正確改動」，風險最低。
