## 5. B2 — line / distance / angle 工具型別（在 `feature/m3-b2` 分支）

### B2a — line 元素
- **Schema v3**：`MeasurementTool.RefToolIds`（複合工具參考元素 Id）。`SchemaVersion` → 3。
- `RecipeRunner` 支援 `line` 工具：ROI → subpix 邊緣 → `FitLine`（M1 `HalconLineFitter`）→ 畫黃色擬合線。line 是元素，**不判定**（IsOk 留白）。
- 配方：`square_edges.zcp`，使用者給出方塊四個角（TL:1447,1059 TR:1447,2059 BR:2647,2059 BL:2647,1059）→ 自動產四條邊的 line ROI。

### B2b — distance 工具（line↔line）
- **兩階段執行**：Pass 1 量元素(line/circle) 並以 Id 索引 → Pass 2 複合工具查 `RefToolIds`。
- distance 工具：`RefToolIds` 取兩 line → `distance_ss` 最小距（`MeasureLineToLine`）→ ×pixel size → mm → 判定。
- 繪製：`DrawDistance`（兩線中點連線 + 數值，OK 綠/NG 紅）。
- 防呆：需正好 2 個、皆存在、皆量測成功、皆為 line。
- 配方：`dist_TB`（高 ~12mm，上↔下邊）+ `dist_LR`（寬 ~10mm，左↔右邊）。

### B2c — angle 工具（line↔line）
- angle 工具：`RefToolIds` 取兩 line → `angle_ll` → **AcuteAngleDeg**（0~90°，用戶選 A，旋轉不變量）。
- 繪製：`DrawAngle`（弧線 + 數值，OK 綠/NG 紅），弧起點由 ref line-A 端點方向算。
- 配方：`ang_TR`（~90°，相鄰邊）+ `ang_TB`（~0°，對邊平行）。
- **bug 修正**：B2c 編輯時 `rows.Add(...)` 被誤刪 → 結果表空白。使用者發現後立即修復。

### GUI 驗證
- 四條邊擬合線貼合、角度 ~90°/~0°、距離 ~10/12mm。OK/NG 依公差正確切換。

**Commit**: `b6092dd` (B2a) / `e200d09` (B2b) / `89d6322` (B2c)
