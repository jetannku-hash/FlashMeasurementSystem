## 2. 修掉的 UI ROI 顯示 bug（2 個）

### Bug A — 第二次以後拖曳看不到藍色 ROI 框
- **檔案**：`App.Wpf/HWindowControlHelper.cs` 的 `Redraw()`
- **根因**：藍框繪製條件被 `_persistentOverlayAction == null` 卡住。第一次量測後 overlay action 被設值，之後拖曳就不畫藍框。
- **修法**：`Redraw()` 開頭加 `_isDrawingRoi` 分支：拖曳新 ROI 期間只畫即時藍框、跳過 stale overlay。

### Bug B — 只有「左上→右下」拖曳會即時顯示藍框
- **檔案**：`App.Wpf/OverlayAnnotator.cs` 的 `DrawRoiRectangle`
- **根因**：`disp_rectangle1` 要求 Row1≤Row2、Column1≤Column2，其他三個拖曳方向違反前提 → 畫出空矩形。
- **修法**：在 `DrawRoiRectangle` 內用 `Math.Min/Max` 正規化座標，四個方向皆可顯示。

1. **持久化**：採 Newtonsoft.Json（M2 才需要；M1 未用到）。

## 7. 4.11 HALCON 數學驗證 → 抓到並修掉 1px 旋轉 bug ⭐

M1 唯一沒被測試覆蓋的真實邏輯是 4.11 的 HALCON 數學。建了臨時 harness（`tests/_CoordVerify/`，references Halcon），用手算案例驗證。

### 抓到的 Bug
`vector_angle_to_rigid` 產生的矩陣，原本用 `affine_trans_pixel` 套用 → **旋轉時 Row 固定差 1px**：
- 中心 (100,100) 旋轉 90° → 落在 (99,100)
- 點距中心 100 → 變 101

探針比對證實正確算子是 **`affine_trans_point_2d`**（引數順序 Row,Col）：中心精確映回 (100,100)、距離精確 100。**手冊提示的 `affine_trans_pixel` 是錯的**（像素中心慣例偏移）。

### 修正
`HalconCoordinateMapper.TransformRoi`：`AffineTransPixel` → `AffineTransPoint2d`，加註解含驗證數據。

### 驗證結果
修正後 harness 全 PASS（恆等/平移/旋轉中心不動/距離保持/垂直/角度）。harness 已**建立→驗證→刪除**，未留正式專案。
