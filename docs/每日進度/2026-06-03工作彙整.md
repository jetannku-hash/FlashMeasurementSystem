## 4. 模板匹配結果 Overlay 修正

### 問題

匹配結果畫面只顯示固定大小的小矩形，無法代表實際模板形狀。

### 根因

原本使用固定尺寸：

```text
DispRectangle2(row, col, angle, 50, 30)
```

這不是 HALCON shape model 的實際輪廓。

### 修正

依 HALCON 官方流程改成顯示實際 shape model contour：

```text
GetShapeModelContours
VectorAngleToRigid
AffineTransContourXld
DispObj
```

### 涉及檔案

- `src/FlashMeasurementSystem.Halcon/TemplateMatching/HalconTemplateMatcher.cs`
- `src/FlashMeasurementSystem.App.Wpf/OverlayAnnotator.cs`
- `src/FlashMeasurementSystem.App.Wpf/MainWindow.cs`

## 5. 旋轉後模板匹配失敗修正

### 問題

使用 `000.png` 建立模板後，`002.png` 中約 20 度旋轉的黑色矩形找不到匹配。

### 根因

- 建模板時 `AngleStart` 被硬寫為 `0.0`，UI / parameter 中設定的 `AngleStart` 沒有生效。
- 匹配時只使用 `TemplateMatchingParameters` 的固定預設角度範圍，沒有使用模型實際建立時的角度範圍。
- 預設建立角度範圍過窄，無法覆蓋 `002.png` 的旋轉角度。

### 修正

- `CreateShapeModel` 改用 `effectiveParams.AngleStart`。
- `LoadModel()` 讀取模型內部 `angle_start` / `angle_extent`。
- `FindShapeModel()` 改用模型實際角度範圍搜尋。
- 預設模板建立角度改為完整旋轉範圍：`AngleStart=-180`、`AngleExtent=360`。

### 驗證

修正後 rotation smoke：

```text
Found=True
Score=0.9995
Angle=20.02°
```

### 涉及檔案

- `src/FlashMeasurementSystem.Domain/TemplateMatching/TemplateCreationParameters.cs`
- `src/FlashMeasurementSystem.Halcon/TemplateMatching/HalconTemplateManager.cs`
- `src/FlashMeasurementSystem.Halcon/TemplateMatching/HalconTemplateMatcher.cs`

## 6. HALCON 顯示視窗 Resize / Zoom / Pan 修正

### 問題

- 主視窗改變大小後圖片會消失。
- 模板匹配綠色 contour 在 zoom / pan / resize 後會消失。
- 需要按滑鼠操作後圖片才會重新出現。

### 根因

- `HWindowControl` resize 後沒有重新 `FitToWindow()` / `Redraw()`。
- 原本 zoom / pan path 各自呼叫 `ClearWindow + DispObj(CurrentImage)`，只重畫圖片，不重畫 overlay。
- 匹配 contour 是一次性繪製，沒有持久化 display state。

### 修正

- 在 `HWindowControlHelper` 建立集中 redraw pipeline：

```text
ClearWindow
SetPart
DispObj(CurrentImage)
Replay persistent overlay
```

- 新增 `SizeChanged` handler，resize 時重新 `FitToWindow + Redraw`。
- zoom / pan / ROI drag 改走集中 redraw。
- 匹配成功後註冊 persistent overlay action，每次 redraw 重新產生並繪製 contour。
- 換圖、匹配失敗、HALCON exception、Dispose 時清除舊 overlay。

### 涉及檔案

- `src/FlashMeasurementSystem.App.Wpf/HWindowControlHelper.cs`
- `src/FlashMeasurementSystem.App.Wpf/MainWindow.cs`

## 7. WinForms 右側控制區 Resize 重疊修正

### 問題

主視窗邊框拖曳或放大後，右側 `Template Matching` 區塊內按鈕、文字、下拉選單與結果框會重疊。

### 根因

- `Template Matching` group box 內部仍使用絕對座標。
- 前一次只調整 `matchResultTextBox` anchor，導致 DPI / font scaling / resize 時 textbox 與上方 controls 的幾何關係不穩。
- 原本右側 panel 缺少完整 layout 容器。

### 修正

- 新增 `NormalizeTemplateMatchingLayout()`。
- 在 `OnLoad()` 時將 Template Matching 既有 controls 放入 WinForms `TableLayoutPanel`。
- 固定上方操作列，結果框佔用剩餘空間。
- `rightPanel.AutoScroll = true`。
- `MainWindow.MinimumSize = 900x500`。

### 涉及檔案

- `src/FlashMeasurementSystem.App.Wpf/MainWindow.cs`
- `src/FlashMeasurementSystem.App.Wpf/MainWindow.Designer.cs`

## 10. 今日結論

今日完成的工作重點是把原本「單點畫面修補」改成更穩定的 HALCON / WinForms 顯示流程：

- HALCON operator 參數依官方文件修正。

- 匹配結果改顯示實際 shape model contour。

後續若要進一步提升操作體驗，建議將主畫面左右區改為 `SplitContainer`，讓 HALCON 影像視窗與右側控制面板有更穩定、可拖曳的空間配置。
