## 二、Phase 2 GUI 驗收期的 6 項修正（commit `d08bc33`）

驗收時發現「純 2D 量測模型配方」被多處只認 1D 工具的舊邏輯擋住，且 1D 結果未標在影像上：

| 主題 | 修正 |
|---|---|
| Metrology 編輯器入口 | 未載入配方時就地用 `Recipe.Default()` 開場，**不需先存一組假 1D 配方** |
| 編輯器另存 | Save 無檔案路徑時跳 **Save As(.zcp)**，編輯器自給自足、不必繞 Edit Recipe |
| RecipeEditor 存檔守門 | 由「0 工具就擋」改為「**有 1D 工具或有量測模型物件**」才算有效 |
| RecipeValidator | 同樣放寬（1D 工具 **或** 量測模型物件）；工具迴圈正規化為非 null 安全略過 |
| 1D line/circle 量測值 | **畫上影像**（比照 distance/angle/metrology），line 值上移避開名稱與亮線 |
| 文字可讀性 | `OverlayAnnotator.DrawText` 改用 HALCON **`disp_text` + 深色半透明底框**，亮特徵上仍清楚（簽章對照 reference 驗證） |

每步 build x64 0/0、Domain 測試綠。
