## 四、⚠️ 並發 agent 事故（本場最重要教訓）

執行批 B 時發現工作樹多出 3 處**非本 session 所寫**的改動——來自 `/resume` 提到、**仍在背景執行的上一個 session agent**，它併發在同一批檔案做重疊的稽核修正：

1. `HWindowControlHelper.GetPersistentOverlayAction()`
2. `MainWindow.cs ~1678` 開/關編輯器 overlay「保存並還原」（#3 的**還原**方案＝使用者**否決**的做法）
3. `MainWindow.cs ~1514` `ResolvePixelSize` 的 `_loadedRecipe != null &&` null guard

**語意衝突**：agent 的 overlay 還原透過 `editor.FormClosed`（後訂閱、後執行）**壓過** RecipeEditor 建構子內先訂閱的 `ClearOverlay()` → 實際行為變成「還原」而非使用者選的「完全清除」。建置能過，但語意被覆蓋。

**處置**（使用者選「保留完全清除、撤掉 agent 改動」）：
- 撤除 ①②（`HWindowControlHelper` `git checkout HEAD`、移除 MainWindow 還原區塊）。
- **刻意偏離使用者「全撤」的一處**：③`ResolvePixelSize` guard **保留**。查證確認它修的是**可觸發真 crash**（載入影像→開 RecipeEditor 未載配方→加 circle→按試測 → `_loadedRecipe.CalibrationProfileId` NRE）。還原 = 重新製造崩潰，故不還原並向使用者說明可反悔。撤除後 build 從 6 warning 降為 0。

**教訓**：多 session／多 agent 併發於同一 repo 時，**動手前後都要 `git status`/`git diff` 逐一核對每個 hunk 的歸屬**；發現非自己所寫的改動先停、surface、確認來源已停止再清理；即使使用者說「全撤」，撞到「還原會製造已驗證 crash」時要 push back 而非盲從。串接 `gsd-execution-lessons`。
