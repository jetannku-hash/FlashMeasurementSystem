## 實機驗收後追加的修正（B 功能）

驗收一鍵量測時用合成測試圖（白底黑形狀）發現連鎖問題，逐一修正：

1. **handler 吞例外 → 看似「沒反應」**：WinForms event handler 內未處理例外不會彈窗。整個 handler 外包 `try/catch (Exception)` 彈出完整訊息；狀態列追加失敗原因。揭露真因 = IQC 早退。
2. **IQC 擋掉合成測試圖**：合成圖過不了真實照片門檻（過亮/飽和/模糊）。決策：`RunOnce` 加 `skipImageQualityCheck` 參數，工具列加 `[略過IQC]` 勾選框（**預設不勾、維持硬擋**，production 行為不變）。
3. **UI 版面重疊**：工具列折成三行蓋住下方欄位。根因是 WinForms docking 依「反 z-order」處理——修正為 `topToolbar.SendToBack()`（Top 先 dock 佔頂）+ `measurementBox.BringToFront()`（Fill 後 dock 佔剩餘）。
4. **殘留匹配綠框**：一鍵流程沒同步匹配狀態回 MainWindow，`DrawRecipeResults` 沿用上次 `_hasMatch` 殘留。修正：`WorkflowResult` 攜帶 `HasMatch/MatchRow/MatchCol/MatchAngleDeg`，繪圖前同步；無參考姿態 → 不畫綠框。實機驗證綠框正確跟隨新影像。
