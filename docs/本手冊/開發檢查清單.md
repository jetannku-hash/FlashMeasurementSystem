# FlashMeasurementSystem 開發檢查清單

本清單配合 `FlashMeasurementSystem_開發手冊.md` 與根目錄 `AGENTS.md` 使用。每次開發前先確認任務類型，再勾選相關項目。

## 1. 實作前

- [ ] 已閱讀 `docs/本手冊/FlashMeasurementSystem_開發手冊.md` 中與本任務相關的章節。
- [ ] 已閱讀即將修改的程式碼與相鄰 pattern。
- [ ] 已確認本次任務是 feature、bugfix、refactor、docs 或 test，沒有混入不相關修改。
- [ ] 已確認是否需要 graphify 輔助。只有大範圍架構理解、依賴分析或手冊對照時才需要。

## 2. 架構與依賴

- [ ] `Domain` 未依賴 UI、Halcon、MES、Reporting、Infrastructure 或檔案系統。
- [ ] `Application` 只表達應用流程與 use case，不直接綁定 UI 或硬體。
- [ ] `Infrastructure` 放基礎設施實作，不反向污染 `Domain`。
- [ ] `Halcon` 放 Halcon 封裝與影像處理適配，不把 Halcon 型別擴散到不該知道 Halcon 的層。
- [ ] `Mes` 放 MES 對接，不與 Halcon/Reporting 互相耦合。
- [ ] `Reporting` 放報表輸出，不承擔量測判定邏輯。
- [ ] `App.Wpf` 只負責 UI 與 composition root。現階段仍可能包含過渡 WinForms 程式碼；若要遷移 WPF，需獨立任務。

## 3. Halcon 與量測規則

- [ ] 專案仍以 .NET Framework 4.8 為目標，除非已明確批准平台遷移。
- [ ] Halcon 相關變更已考慮 x64 執行。
- [ ] Halcon 例外沒有被空白 catch 或靜默吞掉。
- [ ] 影像座標 Row/Column 與物理座標 X/Y 的轉換有明確處理。
- [ ] 需要 UI 顯示的 Halcon 操作在 UI 執行緒執行。
- [ ] 量測流程仍符合手冊狀態機：取像、影像檢查、模板匹配、量測、公差判定、報表/MES、輸出。

## 4. 資料與輸出位置

- [ ] Replay 測試影像放在 `data/images`。
- [ ] `.zcp` 配方檔放在 `data/recipes`。
- [ ] 校正資料放在 `data/calibrations`。
- [ ] CSV 或報表輸出放在 `data/reports`。
- [ ] 不把大型資料、產線輸出、暫存檔或 build artifact 當成原始碼提交。

## 5. 測試與驗證

- [ ] 有對應的單元測試、整合測試或明確的手動驗證方式。
- [ ] 一般變更已執行：

```powershell
dotnet build .\FlashMeasurementSystem.sln /p:GenerateResourceMSBuildArchitecture=CurrentArchitecture /p:Configuration=Debug /p:Platform="Any CPU"
```

- [ ] Halcon 或平台敏感變更也已執行：

```powershell
dotnet build .\FlashMeasurementSystem.sln /p:GenerateResourceMSBuildArchitecture=CurrentArchitecture /p:Configuration=Debug /p:Platform=x64
```

- [ ] 若 build/test 不能執行，已記錄原因、環境限制與替代驗證。

## 6. 完成前回報

- [ ] Summary 說明完成了什麼。
- [ ] Changed files 列出主要變更。
- [ ] Verification 提供實際執行過的命令與結果。
- [ ] Risks 說明已知限制，例如 HALCON 絕對路徑或 transitional WinForms/WPF 狀態。
- [ ] Suggested commit 使用 Conventional Commits，例如 `docs: add project agent rules`。
