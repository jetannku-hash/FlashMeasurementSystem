# 應用落地量測方案庫（Measurement Solutions）— 建議文件

> 產出日期：2026-06-26
> 狀態：**規劃/roadmap，尚未實作**。待幾何基元(A3)與校正(A1)等底層備齊後，再拿出來逐項討論實現順序。
> 來源討論：使用者觀察到「目前都在做幾何基元(工具)，但現場操作員要的是『選一個應用方案 → 對準工件 → 直接輸出數值+判定+報告』」。本文件把這個方向具體化。

---

## 1. 核心概念：基元(工具) → 方案(應用)

| 層級 | 內容 | 使用者要做什麼 |
|------|------|----------------|
| **幾何基元(已做)** | edge / line / circle / ellipse / rectangle / arc 卡尺 | 自己決定跑哪個、算什麼、怎麼判定 ← 門檻高 |
| **量測方案(本文件)** | 具名的「落地檢測項目」 | 選方案 → 框特徵 → 設公差 → 一鍵輸出 ← 現場可用 |

**方案 = 編排(基元 → 計算 → 判定 → 輸出)的具名模板。** 操作員不碰底層 operator。

### 與現有架構的關係（已有半套地基）
專案已有 `Recipe`、`MeasurementTool`、`RecipeRunner`、`RecipeEditor`、`ToleranceJudger`、CSV 報表。
「量測方案」= 在這之上加一層 `MeasurementSolution`：
- 一個具名模板，內含「用哪些基元 + 計算邏輯 + 公差欄位 + 輸出格式」。
- 操作員從下拉選方案 → 套到當前影像/ROI → 得到 結果值 + PASS/FAIL + 報告列。
- 技術上可實作為「預設配方範本 + 對應的分析器(Analyzer)」。

### 每個方案的統一規格（實作時每項都要定義）
1. **檢測對象**（量什麼工件特徵）
2. **底層基元**（呼叫哪些已實作功能）
3. **計算輸出**（從基元結果算出的量測值）
4. **判定**（公差/規則 → PASS/FAIL）
5. **輸出**（數值表 + 判定 + 報告/MES 欄位）
6. **前置需求**（是否需校正、樣板對位、或尚未實作的新基元）

## 2. 方案庫（依領域）

> 標記：✅=底層基元已具備，可直接編排；🔶=需先補小功能(如去重/統計)；🆕=需新增基元/能力。

### 2.1 機械加工件 / 正軟件
| 方案 | 檢測對象 | 底層基元 | 計算輸出 | 標記 |
|------|----------|----------|----------|------|
| 齒輪齒數/齒距 | 齒輪外齒 | arc | 齒數、齒距均勻度、缺齒/斷齒 | 🔶(去重+統計) |
| 螺孔圓 PCD | 法蘭螺孔陣列 | arc + circle | 孔數、各孔角度、PCD 直徑、孔位偏差 | 🔶 |
| 軸徑/孔徑 | 圓柱軸、內孔 | circle | 直徑(mm)+ 公差 | ✅(需校正) |
| 真圓度/同心度 | 軸、孔 | circle×N | 圓度、同心度 | 🔶(多圓分析) |
| 鍵槽寬度/位置 | 軸上鍵槽 | edge pair / rectangle | 槽寬、對中心角度 | ✅ |
| 倒角/R 角 | 邊角圓弧 | arc 擬合(circle) | R 值、起訖角 | ✅ |

### 2.2 電子 / PCB / IC
| 方案 | 檢測對象 | 底層基元 | 計算輸出 | 標記 |
|------|----------|----------|----------|------|
| 引腳間距(pitch) | IC/連接器引腳 | edge(多邊) | 相鄰引腳間距、均勻度、缺腳 | 🔶(多邊統計) |
| 引腳寬度/共面 | 引腳 | edge pair | 腳寬、偏移 | ✅ |
| 鍍通孔/焊盤對位 | PCB 孔、pad | circle + 樣板對位 | 孔徑、孔對 pad 偏心 | ✅(需對位) |
| 元件位置/旋轉 | 貼片元件 | 樣板比對 | 位置偏移、旋轉角 | ✅(已有 matching) |
| 金手指間距 | 連接器金手指 | edge(多邊) | 間距、寬度 | 🔶 |

### 2.3 衝壓 / 成型件
| 方案 | 檢測對象 | 底層基元 | 計算輸出 | 標記 |
|------|----------|----------|----------|------|
| 外形輪廓尺寸 | 衝壓件外框 | rectangle / line×N | 邊長、對角線、垂直度 | ✅ |
| 孔位/孔徑陣列 | 衝壓孔群 | circle×N + 對位 | 各孔徑、孔距、孔位偏差 | 🔶 |
| 開口/缺口寬度 | 沖切缺口 | edge pair | 缺口寬、深 | ✅ |
| 折彎角度 | 折彎邊 | line×2 | 夾角 + 公差 | ✅ |
| 毛邊/缺料偵測 | 邊緣異常 | 輪廓比對 | 缺陷面積/位置 | 🆕(需輪廓比對) |

### 2.4 通用（不分領域，最常用）
| 方案 | 檢測對象 | 底層基元 | 計算輸出 | 標記 |
|------|----------|----------|----------|------|
| 兩點/兩邊距離 | 任意兩特徵 | edge/line×2 | 距離(mm)+ 公差 | ✅(需校正) |
| 寬度/間隙 | 平行兩邊 | edge pair | 寬度 + 公差 | ✅ |
| 兩邊夾角 | 任意兩邊 | line×2 | 角度 + 公差 | ✅ |
| 平行度/垂直度 | 兩邊關係 | line×2 | 偏差角 | 🔶 |
| 圓直徑 | 任意圓特徵 | circle | 直徑 + 公差 | ✅ |

## 3. 市面常見影像量測儀的應用方案（使用者要求補充）

主流影像量測儀(Keyence IM、Mitutoyo QV、OGP 等)出貨時內建的「一鍵量測項目」，可作為本專案方案庫的對標清單：

- **尺寸量測**：長度、寬度、距離、孔徑、軸徑、間隙、深度。
- **圓/弧**：直徑、半徑、圓度、同心度、圓弧角度。
- **角度**：兩線夾角、平行度、垂直度、傾斜度。
- **位置度/GD&T**：真位置度、輪廓度、跳動。
- **計數/陣列**：孔數、齒數、引腳數、間距均勻度（← arc / 多邊統計）。
- **輪廓比對**：與標準輪廓(CAD/master)比對找偏差、缺料、毛邊。
- **樣板定位 + 自動量測**：先 find_shape_model 對位，再批次套用所有量測項（本專案已有對位基礎）。
- **批次/SPC 輸出**：CSV/報表、趨勢圖、GR&R（手冊 §6.3）。

## 4. 建議落地順序（待討論，非定案）

1. **先把 arc 做成第一個完整方案範本**：齒數/齒距(去重+統計+判定) → 驗證「基元→方案」這條 pipeline 走得通。
2. **通用三方案**(距離 / 孔徑 / 角度)：覆蓋面最廣、依賴最少(需校正 A1)。
3. **多特徵統計類**(引腳 pitch、孔位陣列)：補「多邊/多圓統計 + 均勻度」共用分析器。
4. **GD&T / 輪廓比對**：對標高階量測儀，需 A4(形位公差)、新輪廓比對基元。

每項落地前依統一規格(§1)定義清楚，並沿用 feature-adapter 模式。

## 5. 先決條件提醒
- 多數「尺寸類」方案要先有 **A1 像素→mm 校正**，否則只能輸出像素值。
- 「自動批次量測」要先有 **樣板對位**(已具備 `find_shape_model` + `vector_angle_to_rigid`)。
- 「輪廓比對/缺陷」需 **新增輪廓比對基元**(目前沒有)。
