# 引腳間距(pin-pitch)量測方案 — 執行計畫

Phase 5 命名方案庫第三個工具(前有 gear、PCD)。分支 `feature/pin-pitch`(做前開)。
用 subagent-driven-development 逐 task 執行:每 task 實作 → 建置/測試 → 規格審 → 品質審 → commit。

## 設計決定(已敲定)
- `ToolType = "pin_pitch"`。schema **v12**。
- **幾何**:重用既有 `RoiGeometry` rect2(`tool.Roi`),**非** ArcRoi(pins 是直排,不是放射/環形)。ROI 由使用者在影像上拖矩形框住一排腳。
- **偵測**:Blob(重用 PCD 手法)。新 `IPinDetector.DetectPinsInRect(image, RoiGeometry, params)` + `HalconPinDetector`:rect2 `reduce_domain` → `binary_threshold(PinIsDark?"dark":"light")` → `connection` → `select_shape(area, MinPinAreaPx…)` → `area_center` → 腳形心。`EnsureSingleChannel` 保護。
- **極性**:背光(亮底暗腳),`PinIsDark=true` 預設;欄位可配置。
- **分析**(純 Domain `PinPitchAnalyzer`):≥2 腳;對形心擬合一條線(取主軸方向)→ 投影排序 → 相鄰間距(px→mm 用 pixelSizeUm)→ 平均間距、最大單間距偏差、腳數、垂直於擬合線的位置偏差(直線度);缺腳 = 某間距 > 1.5×中位間距。
- **四判定**:①腳數(count==Nominal)②平均間距(|mean−NominalPitch|≤tol)③均勻度(maxDev≤tol)④缺腳(無異常間距)。`IsPass = 全 OK`。CSV 四列,比照 gear/PCD 自發 ItemJudgment(不走 GetMeasuredValue,避免假 NG)。
