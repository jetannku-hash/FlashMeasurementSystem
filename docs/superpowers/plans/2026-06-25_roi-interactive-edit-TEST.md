# ROI 互動編輯（rect2）手動驗收測試步驟

- 日期：2026-06-25
- 前提：`Rect2EditMathTests` 單元測試已通過，Any CPU / x64 建置 0 error 0 warning
- 環境需求：HALCON 17.12 license + 一張測試影像（至少 800×600）

---

## 測試 1：畫完 ROI 自動出現綠色編輯框

| # | 操作 | 預期結果 |
|---|------|----------|
| 1 | 啟動 App，切到 **Edge Detection** 分頁 | |
| 2 | Load Image → 選一張 .png/.jpg | 影像顯示在視窗中 |
| 3 | 勾選 **Draw ROI** checkbox | |
| 4 | 在影像上拖曳一個矩形 ROI | 放開左鍵後：**藍色矩形消失**，取而代之出現**綠色矩形外框 + 8 個白色方塊把手 + 一條連線接到圓形旋轉鈕** |
| 5 | 確認 8 個把手位置 | 4 角 + 4 邊中點（上下左右），全在綠色框線上 |
| 6 | 確認旋轉鈕 | 在長軸方向（角度方向）延伸出去，有連線桿連接 |

## 測試 2：拖曳移動（Body）

| # | 操作 | 預期結果 |
|---|------|----------|
| 1 | 在綠色框**內部**（非把手）按下滑鼠左鍵 | |
| 2 | 拖曳到新位置 | 整個綠色框 + 把手 + 旋轉鈕**一起平移** |
| 3 | 放開滑鼠 | 框停在最後位置，**數值框 ScanLength/Width 不變，Angle 不變**（僅位置改變） |
| 4 | 觀察三個數值框 | 只有 ROI 位置反映到 UI？實際上此功能尚未實現中心位置數值框（CenterRow/CenterCol 目前無直接數值框），以 `_latestEdgeRoi` 內部更新為準 |

---

## 測試 3：拖曳角把手縮放（對稱繞中心）

| # | 操作 | 預期結果 |
|---|------|----------|
| 1 | 拖曳**任一角**的把手方塊 | |
| 2 | 向外拖 | 框**對稱放大**（對面角也向外），Scan Length 和 ROI Width 數值框**同時即時更新** |
| 3 | 向內拖 | 框對稱縮小，最小值不會低於 3 pixel（MinHalfLen） |
| 4 | 放開 | 數值框停在最終值 |
| 5 | 重複操作另三個角 | 行為一致 |

---

## 測試 4：拖曳邊中點把手縮放（只改一邊）

| # | 操作 | 預期結果 |
|---|------|----------|
| 1 | 拖曳**長軸端點**（左右）把手 | 只改 **Scan Length**（l1），ROI Width 不變 |
| 2 | 拖曳**短軸端點**（上下）把手 | 只改 **ROI Width**（l2），Scan Length 不變 |
| 3 | 觀察對稱性 | 每次改變都是**對稱繞中心**（對面把手同步反向移動） |

---

## 測試 5：旋轉鈕改角度

| # | 操作 | 預期結果 |
|---|------|----------|
| 1 | 拖曳**圓形旋轉鈕**（綠色框延伸出去的圓點） | |
| 2 | 繞中心轉圈 | 綠色框 + 所有把手**同步旋轉**，旋轉鈕始終在長軸延伸方向 |
| 3 | 觀察 Angle 數值框 | 角度值（度）**即時更新**，範圍約 -180° ~ +180° |
| 4 | 放開 | Angle 停在最終值 |

---

## 測試 6：數值框 → 把手同步（反向）

| # | 操作 | 預期結果 |
|---|------|----------|
| 1 | 綠色編輯框顯示中，手動改 **Scan Length** 數值（例如 100 → 80） | 綠色框的長軸**立刻**縮短為 80 |
| 2 | 改 **ROI Width**（例如 50 → 30） | 綠色框的短軸立刻縮短 |
| 3 | 改 **Angle**（例如 0 → 45） | 綠色框立刻旋轉到 45° |
| 4 | 連續快速改三個值多次 | **無閃爍、無無限迴圈**、數值不回彈 |
| 5 | 在數值框輸入值，再拖曳滑鼠 | 滑鼠改動**回寫數值框**無誤差（度 ↔ 弧度轉換一致） |

---

## 測試 7：斜框 Detect（核心驗證）

| # | 操作 | 預期結果 |
|---|------|----------|
| 1 | 載入有明顯斜邊的影像（例如 45° 邊緣） | |
| 2 | 畫 ROI 後，拖曳旋轉鈕讓**綠色框長軸對齊斜邊**（例如 ~45°） | |
| 3 | 按 **Detect** | 量測發生在**綠色框的方向**（不是鏡像方向）；斜邊上偵測到 edge points |
| 4 | 在 Edge Results Grid 中檢查 edge point 數量 | 邊緣點集中在斜邊沿線（row/col 分佈合理） |
| 5 | 按 **Fit Line** | 擬合線角度 ≈ ROI 角度（誤差 < 幾度），**不是** 90° 或 0° |
| 6 | 對比：角度設 0° 按 Detect → 再設 ~45° 按 Detect | 兩次 edge point 分佈方向完全不同（證明角度有吃進去） |

## 測試 8：放大縮小後把手恆定

| # | 操作 | 預期結果 |
|---|------|----------|
| 1 | 先畫一個 ROI 進入編輯模式（綠色框出現） | 記住把手大小（方塊視覺尺寸） |
| 2 | 用滑鼠滾輪**放大影像**（zoom in） | 把手在**螢幕上**大小不變（方塊在影像座標中變小、但像素恆定） |
| 3 | 滾輪**縮小影像**（zoom out） | 把手在螢幕上大小仍不變 |
| 4 | 放大後拖曳把手 | 命中判定範圍依然正確（不會因放大而難以點到） |

## 測試 9：模板比對不受影響

| # | 操作 | 預期結果 |
|---|------|----------|
| 1 | 切到 **Inspection** 分頁（Template Matching） | |
| 2 | Load Reference Image → 勾選 ROI Mode → 畫矩形 | 藍色正交框正常顯示（非綠色） |
| 3 | 按 **Create Template** | 模板建立成功，無錯誤 |
| 4 | Load Test Image → 選模板 → **Run Matching** | 匹配結果正常顯示（綠色/紅色 Match 框 + score） |
| 5 | 切回 Edge Detection 分頁 | 綠色編輯框還在（兩分頁獨立） |
| 6 | 反覆切換分頁數次 | 無當機、無 overlay 殘留 |

## 測試 10：Recipe Editor 互動編輯

| # | 操作 | 預期結果 |
|---|------|----------|
| 1 | 先載入影像，按 **Edit Recipe** 開啟編輯器 | |
| 2 | 按 **+ Circle** 新增一個 circle 工具 | 編輯器右側出現 ROI 數值框，**主畫面上的影像同時出現綠色把手** |
| 3 | 在影像上拖曳圓形工具的把手 | 編輯器數值框即時更新，Title 出現 `*` dirty 標記 |
| 4 | 手動改編輯器的 Angle (rad) 數值 | 影像上的綠色框同步旋轉 |
| 5 | 按 **+ Line** 再新增 line 工具 | 切換到 line，影像上的框更新為 line 的 ROI |
| 6 | 按 **Take ROI from Image** | 在主畫面拖曳新 ROI，放開後**自動進入編輯模式（綠色把手出現）** |
| 7 | 在 Recipe Editor 中選 distance 或 angle 工具 | 綠色把手**消失**（composite tool 不支援編輯） |
| 8 | 關閉 Recipe Editor（按 X） | 回到主畫面，綠色把手消失 |
| 9 | 再次選 circle/line 工具 | 綠色把手再次出現 |

## 測試 11：邊界條件與防呆

| # | 操作 | 預期結果 |
|---|------|----------|
| 1 | 把 ROI 縮到最小（向內拖到底） | 框不消失，最小半長 = 3 px（MinHalfLen） |
| 2 | 未載入影像時按 Detect | 顯示 "Please load an image first"（無 NullReferenceException） |
| 3 | 未畫 ROI 時按 Detect | 顯示 "Please draw an ROI region first"（基於 `_latestEdgeRoi == null`） |
| 4 | 在編輯模式中按滑鼠右鍵平移影像 | 正常平移（不觸發編輯行為） |
| 5 | 滾輪縮放時 | 不觸發編輯行為，僅縮放 |
| 6 | 換圖 → 重新畫 ROI | 舊編輯框清除，新編輯框出現 |
