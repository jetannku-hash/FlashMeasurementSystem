## 二、能力差距清單（Tier A：量測能力核心）

### A1. 完整相機校正 + 鏡頭畸變 + world-plane 度量
- **功能介紹**：用標定板（caltab）做相機校正，求得內參、外參與鏡頭畸變係數，再把量測點／影像投影到 z=0 世界平面，得到**全視場一致、去畸變的真實 mm 值**。取代現在「單一等向 pixel size」——後者在視場邊緣、鏡頭桶形畸變下會 silently 偏差。這是主流量測儀「準確度」的根。
- **HALCON operator**：`create_calib_data`, `find_calib_object`, `calibrate_cameras`, `get_calib_data`, `set_calib_data`, `gen_image_to_world_plane_map` + `map_image`（去畸變整圖）, `image_points_to_world_plane`（量測點轉世界座標）。
- **工作量**：完整版 ~8–14 天；**軟體先行半套**（兩正交標準件各校一次 → 非等向 X/Y pixel size，不靠 caltab）~3–4 天。
- **難易度**：**高**。HALCON calib data model 複雜、座標慣例（相機/世界/影像）多，易接錯。
- **風險**：**高**。需實體 caltab 並拍多張校正影像；本機無硬體則無法驗證；座標系接錯會「不報錯但量錯」。建議先做非等向半套止血，畸變校正等 caltab 到位再做。

### A2. 2D Metrology Model（量測模型）
- **功能介紹**：在名義幾何上自動沿邊佈設模糊量測矩形，一次套用到影像、穩健擬合出 line/circle/ellipse/rectangle 並回傳參數與量測點。比現在「手動放 ROI → 抓邊 → 擬合」更**自動、抗噪、可重複**，是主流量測儀的「量測模型」核心，能一鍵量整個工件多特徵。
- **HALCON operator**：`create_metrology_model`, `set_metrology_model_image_size`, `add_metrology_object_generic`（或 `add_metrology_object_line_measure` / `_circle_measure` / `_ellipse_measure`）, `set_metrology_object_param`, `apply_metrology_model`, `get_metrology_object_result`, `get_metrology_object_measures`, `get_metrology_object_model_contour`。
- **工作量**：~10–16 天（新 feature adapter + Domain 模型 + GUI 佈設介面 + 與既有 recipe 整合）。
- **難易度**：**中高**。operator 成熟，難在 GUI 佈設體驗與既有 ROI/recipe 模型整合。
- **風險**：**中**。可與現有 1D 並存、不破壞既有流程；主要風險是設計上要不要讓它「取代」還是「並列」現有量測管線。

### A3. 幾何基元擴充（橢圓 / 圓弧 / 矩形 / 點）
- **功能介紹**：補上橢圓、圓弧、矩形、點的量測與擬合，涵蓋長圓孔、倒角、鍵槽、R 角等特徵。現只有線 + 圓。
- **HALCON operator**：`fit_ellipse_contour_xld`, `gen_measure_arc`（圓弧卡尺）, `fit_rectangle2_contour_xld`。
- **工作量**：每基元 ~2–4 天，全套 ~6–10 天。
- **難易度**：**低中**。直接沿用 `HalconCircleFitter` 的 feature-adapter 範式照抄。
- **風險**：**低**。增量、隔離、易單元測試。`fit_ellipse` 最低風險；`gen_measure_arc` 需留意角度/半徑參數慣例（查 reference 再寫）。

### A4. GD&T 形位公差
- **功能介紹**：真圓度、真直度、平行度、垂直度、同心度、位置度等形位公差判定，而非只有尺寸 ±。含基準（datum）系統。主流量測儀標配。
- **HALCON operator**：多為 Domain 數學（由已擬合幾何計算）；HALCON 輔助 `roundness`，構造用 `intersection_lines`, `distance_pl`, `angle_ll`。
- **工作量**：~8–14 天（定義 datum、公差類型、判定數學、報表欄位）。
- **難易度**：**中高**。難在 GD&T 語意與基準系統正確性，不在 operator。
- **風險**：**中**。算錯不報錯但判定錯，需大量對標（與既有 CMM/標準件）驗證。建議逐項目交付、每項配 failing test。

### A5. 幾何構造運算
- **功能介紹**：線交點、對稱中線、點到線投影、切線、最佳擬合基準，讓量測項可組合（例：兩線交點到圓心距離）。現有距離/角度已部分覆蓋，缺構造幾何。
- **HALCON operator**：`intersection_lines`, `distance_pl`, `angle_ll`（投影/中線可用 Domain 數學）。
- **工作量**：~5–8 天。
- **難易度**：**中**。
- **風險**：**低中**。多為純數學，易單元測試。

## 三、能力差距清單（Tier B：穩健性與量產可用性）

### B1. Fuzzy / robust 自動邊緣量測
- **功能介紹**：用模糊權重（對比、位置、形狀）挑正確邊，抗雜訊、反光、干擾邊，提高量產穩定度。
- **HALCON operator**：`fuzzy_measure_pos`, `fuzzy_measure_pairs`, `set_fuzzy_measure`。
- **工作量**：~3–5 天（擴充現有 `HalconEdgeDetector` 為進階模式）。
- **難易度**：**中**。 **風險**：**低**（與現有 measure_pos 並存）。

### B2. GR&R / 重複性自助工具
- **功能介紹**：重複跑同一配方 N 次，算 6σ、重複性/再現性%，評估量測系統能力。手冊 §6.3 已描述。
- **HALCON operator**：無（純統計）。
- **工作量**：~3–5 天。 **難易度**：**低**。 **風險**：**低**。

### B3. 報表（PDF）+ MES 整合
- **功能介紹**：PDF 量測報表 + MES 上傳（現只有 CSV）。
- **HALCON operator**：無（整合）。
- **工作量**：PDF ~3–5 天；MES skeleton ~6–10 天。
- **難易度**：**低中**。 **風險**：MES 卡外部協議規格（需使用者提供）。

### B4. 自動對焦（Z 軸）
- **功能介紹**：自動對焦取得最清晰影像。**需 Z 軸馬達硬體，本機無法實作/驗證**。列出待硬體到位。

## 四、建議優先順序（量測能力影響 × 軟體可先行）

1. **A3 幾何基元擴充**（低風險、快見效、補特徵廣度）
2. **A1 校正半套（非等向 X/Y）**（直接提升所有 mm 準確度，不需硬體）
3. **A2 2D Metrology Model**（自動化/可重複性，主流量測儀的分水嶺）
4. **A5 幾何構造 → A4 GD&T**（撐起形位公差能力）
5. **B1 fuzzy 量測 → B2 GR&R**（量產穩定度與系統能力證明）
6. **A1 畸變校正完整版 / B4 自動對焦**（等 caltab / Z 軸硬體）
7. **B3 PDF / MES**（等 MES 協議規格）

> 註：工作量為單人粗估，含測試但未含需求釐清、對標驗證與硬體取得。每項落地前應先依 `AGENTS.md` 與開發手冊確認規格，並對 reference 核 operator 參數順序。
