## 二、各項具體做法摘要

### 🟢 4.8 公差判定（P1）
- 純比較邏輯，不碰 HALCON。實測 vs nominal±tol → OK/NG + 偏差 + 接近邊界警告。
- 手冊的 `margin = |deviation|/(upper-lower)` 建議改為「到最近邊界的餘量百分比」，語意較準。

### 🟢 4.11 座標系定位（P1）
- KB 依據：`vector_angle_to_rigid.txt`、`affine_trans_pixel`。
- **不要照手冊手刻旋轉矩陣**（它假設「參考原點在 (0,0)」會錯且重造輪子）。改用 HALCON `hom_mat2d` + `vector_angle_to_rigid` + `affine_trans_pixel`（專案 `HalconTemplateMatcher` 已這樣用）。

### 🟢 4.13 尺寸標註（P3）
- **擴充現有 `OverlayAnnotator`**，補 `DrawDistance` / `DrawAngle` / `DrawResultTable`。不要另開 `DimensionAnnotator` 跟現有重複。
- 手冊範例 `DrawRoi` 少一個 `}`，照抄會編譯失敗。

### 🟢 4.12 ROI 編輯（P2）
- 缺的是「多工具 ROI 資料模型 + 每工具參數 + 持久化（配方）」。
- 手冊的 `MeasurementRoi` 把幾何 + 邊緣參數 + 公差 + 結果全塞一類（胖 DTO），建議依分層拆開。

### 🟡 4.10 校正管理（P2，僅簡易版）
- 簡易版：兩點已知距離 → 算 pixel size，取代目前寫死的 `10.0 µm/px`。只需一個已知尺寸標準件，不需馬達。
- 完整版（畸變校正）暫不做：需 HALCON 標定板。
- 手冊「斜向校正 `pixelSize/cosAngle`」邏輯可疑，建議 X、Y 各用一條正交已知距離分別校。

### 🔴 4.9 自動對焦（暫不做）
- 評分部分（Laplacian 變異數 / KB `determine_sharpness` 四指標）可做且可單元測試；閉迴路需 Z 軸馬達硬體。本輪不做。
